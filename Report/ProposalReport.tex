\begin{abstract}
In this report, it is proposed to utilize an offline reinforcement learning program in conjunction with one or more basic path planning algorithms to efficiently optimize a simulated turtle-bot travel path in a 2D environment based on the manipulation of several key parameters with the performance measured against path length, smoothness, travel time, velocity, and clearance.
\end{abstract}

\section{Introduction}
In the industrial environment of AMRs, path planners are, in general, deployed en-mass and are not tailored to the customers needs or the warehouse environment in which they are deployed. This results in good, but not excellent performance, and while easier to maintain on the backend, can limit customer output in on the front end. These general deployments often cast a wide net and aim to operated on the 'safe side', whether that be wide, nonadjustable clearance, paths around high traffic areas, or creating lanes that avoid foot traffic altogether. In reality, each customer has different needs, and outside of required safety standards, may have certain workplace standards that could offer better performance if utilized properly to tune the AMR deployment. It is obvious that hand tuning any number of parameters for each individual deployment site is unreasonable and the time spend on it would far outweigh any long-term benefits.
To reduce the amount of work input needed to tune these parameters, it is proposed that a reinforcement learning (RL) algorithm is deployed for each site and the parameters are adjusted with direct input of the customer to optimize the AMR deployment for their specific site while prioritizing their specific needs. The evaluation metrics to be utilized in this simulation are: Path length, path smoothness, travel time, clearance, and average velocity.
    \subsection{Deliverables}
    At the end of the semester, the team will have a functional program that runs a 2D simulation utilizing a Turtlebot which needs to navigate through a crowded warehouse-like environment utilizing one or multiple path planners in conjunction with a RL algorithm that will automatically tune the path and path parameters to optimize the path of travel to align with the request. In this simulation, getting from point A to point B the fastest will not be the only measure of success. The RL algorithm should be able to take in parameter adjustments to the desired path, acceptable smoothness, desired clearance, and zone restrictions and create an optimal plan for the turtlebot.

                \subsubsection{Comparative Analysis of Local Trajectory Planning Algorithms in ROS2}
        In this paper, Elbouhy et. al designed a study that consisted of both a simulation and real world test of a small mobile robot similar to an AMR utilizing ROS2 and three different path planners. The path planners were tested in both environments with static and dynamic obstacles and the results were then evaluated to determine the different in approach for each algorithm. The study analyzed a DWB (Modified Dynamic Window Approach), MPPI (Model Predictive Path Integral), and RPP (Improved Pure Pursuit) planner and used the following criteria to determine path quality: time, distance, smoothness, minimum clearance, runs completed, and number of recoveries (if obstacle contact is made). After running 10 runs of each planner in both the simulation and real environments, it was clear that MPPI was the fastest but least smooth, DWB operated with the least clearance, and RPP took the most time and traveled the longest distance.

        As this relates to this project, this clearly demonstrates that, as expected in some cases, different planners have different strengths and weaknesses. One of the potential goals for this project is to utilize multiple path planners in concert with the RL algorithm and the research conducted in this paper shows that an approach like that could, in some circumstances, optimize a path. It also shows that if certian critera are considered higher value, then the RL could favor those. For example, if the environment is close quarters and completely static, DWB could be favored for it's low clearance and shorter distance traveled.
        
        \subsubsection{Local Path Planning Algorithm of Mobile Robot by Modified Optimized Parameters Method (2010)}
        In this paper, Li et. al studied the Dynamic Window Approach (DWA) algorithm and adjusted specific evaluation parameters in an effort to optimize the path planning sequence. DWA is a local path planner that, in this case, works in concert with a global planner. DWA measures and simulates velocity values for the robot over a specific period of time, then with that, it generates and evaluates the paths created and determines the optimal choice. The evaluation function utilized in this paper can be seen below where heading is the difference in angle between the current and goal state, dist is the distance between the trajectory and nearest obstacle, and velocity is the speed of the robot, and $\alpha$, $\beta$, and $\gamma$ are the tuning values under study.
        \[ G(\upsilon,\omega) = \sigma(\alpha \cdot heading(\upsilon,\omega) + \beta \cdot dist(\upsilon,\omega)+\gamma \cdot velocity(\upsilon, \omega))\]
        A simple environment was set up in both a simulation and real-world and $\alpha$, $\beta$, and $\gamma$ values were adjusted  as follows for 3 separate preliminary trials: (0.05, 0.10, 0.10), (0.10, 0.20, 0.20), (0.15, 0.30, 0.30). And it was determined that $\alpha$ had a minimal effect on optimizing the path and was set to 0.10 for the rest of the experiment. $\beta$ and $\gamma$ were then tuned between 0.05 and 0.4 in a series of 64 combinations to optimize both at $\beta$ = 0.04 and $\gamma$=0.36. 

        In the context of this project, the interest lies in the tuning equations itself. With the goal of this project being to implement a RL-supported algorithm for the path planning, the tuning equation above provides not only quality reference for this project, but also acts as a confident proof-of-concept in that it is known that path planning algorithms can be tuned and optimized on a limited number of parameters and the results are statistically significant and positive.
